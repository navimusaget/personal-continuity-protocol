# Relay Agent Guide

**Draft implementation guide - not legal advice.**

The Core Paper separates witnesses, validators, stewards and relay agents because role confusion creates capture.

## Roles

### Witness Node

A Witness Node records limited event evidence:

- hash;
- timestamp;
- receipt;
- event type;
- dispute or revocation marker;
- pointer to a private, material or legal package.

A witness does not certify the whole person.

### Validator

A Validator checks formal sufficiency for a defined process. For example, a platform, notary, legal agent or foundation may check whether a continuity notice contains the required fields.

A validator may reject, request correction or escalate. It should not become the owner of continuity.

### Relay Agent

A Relay Agent helps a subject or authorized representative move from silence or rupture back into review.

A relay agent may:

- help locate event handles;
- contact witness nodes;
- notify platforms, publishers, foundations or legal agents;
- request protected pause;
- help identify material anchors;
- help assemble an evidence bundle;
- help route the subject toward qualified counsel or appropriate institutions.

A relay agent must not:

- acquire authority merely by relaying;
- transfer assets;
- release archives;
- declare succession;
- certify identity beyond what it can actually verify;
- demand unnecessary biography;
- keep the subject dependent on one relay.

## Core rule

A witness records.  
A validator evaluates.  
A relay transmits and assists.

Relaying is not authority.
