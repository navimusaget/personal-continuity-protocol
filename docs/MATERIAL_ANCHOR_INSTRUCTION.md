# Material Anchor Instruction

**Draft instruction - adapt to local law and archival practice.**

A digital continuity proof needs a material witness.

A domain can be lost. A platform can close. A cloud account can disappear. A database can be deleted. A QR code can rot behind a login. A court may not trust a self-made file without a reliable path.

The Material Anchor gives the protocol a body.

## Recommended material package

A basic material anchor may include:

- printed Continuity Fingerprint Sheet;
- package title and version;
- SHA-256 hashes of digital evidence package and court-readable PDF;
- event handles;
- list of public witness records;
- date of creation and review;
- subject declaration or signature where appropriate;
- witness signatures where appropriate;
- notarial deposit reference if available;
- physical medium reference if included;
- human-readable recovery instructions.

## Storage pattern

High-risk packages should not rely on one material copy.

Consider distributing copies across different classes:

1. subject-held copy;
2. legal professional or notary copy;
3. archive, library or university copy;
4. trusted witness copy;
5. durable hash/fingerprint stored separately from sealed contents.

## Good practice

Use:

- tamper-evident sealing;
- version numbers;
- chain-of-custody notes;
- witness signatures;
- public hashes before dispute;
- periodic re-deposit after important changes.

## Bad practice

Do not store private keys, passwords, raw seed phrases or sensitive archives in an unsealed public package.

Do not let a single custodian become the whole continuity.

## Core rule

The last mile is not an app.

The last mile is a thing that survives silence.
